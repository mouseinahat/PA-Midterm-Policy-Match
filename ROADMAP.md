# Pennsylvania Policy Match

A privacy-first policy matching platform that helps voters understand where they align on public policy, while laying the foundation for a broader civic opinion and consensus platform.

---

# Vision

Most political platforms focus on candidates.

We focus on policies.

Our long-term goal is to build an open, transparent, and privacy-preserving system that helps citizens:

* Understand their policy preferences
* Compare themselves with candidates
* Explore public issues
* Participate in civic discussions
* Contribute to trustworthy public opinion statistics

Without requiring personal identity disclosure.

---

# Current Status

## v0 (Completed)

### Pennsylvania Governor Match

Features:

* Pennsylvania 2026 Governor Match
* Josh Shapiro vs. Stacy Garrity
* 25 policy questions
* 12 policy dimensions
* ZIP Code routing prototype
* Methodology page
* Privacy page
* GitHub Pages deployment

Limitations:

* Candidate vectors are manually modeled
* ZIP mapping is only a sample dataset
* No database
* No user accounts
* No anti-bot protections
* No public statistics

# Product Roadmap

## v1 — Source-Based Candidate Matching

Goal:
Improve credibility and transparency.

Features:

* Candidate position citations
* Campaign website references
* Position evidence viewer
* Improved scoring methodology
* Expanded Pennsylvania issue coverage

Success Metric:

* Every candidate position has at least one public source

---

## v2 — Real Pennsylvania Ballot Engine

Goal:
Move from statewide matching to actual ballot matching.

Features:

* Real ZIP Code lookup
* Congressional district detection
* House candidate matching
* State legislature support
* Local election support

Success Metric:

* User sees the candidates that actually appear on their ballot

---

## v3 — Privacy-Preserving User Profiles

Goal:
Create personalized experiences without collecting personal identity.

Features:

* Anonymous participation keys
* Local-only policy profile storage
* Political DNA profile
* Result sharing
* Policy preference history

Principles:

* No names
* No phone numbers
* No government IDs
* No political profile storage on servers

Success Metric:

* Returning users can view their previous results without creating an account

---

## v4 — Civic Statistics Platform

Goal:
Collect issue-level opinion data.

Features:

* Issue polling
* Anonymous responses
* Public dashboards
* Opinion trend tracking
* Open aggregate statistics

Success Metric:

* Citizens can explore real-time issue opinions across Pennsylvania

---

## v5 — Anti-Manipulation Infrastructure

Goal:
Protect statistics from spam and coordinated manipulation.

Features:

* Anonymous participation keys
* Cloudflare Turnstile
* Rate limiting
* Suspicious activity detection
* Trust-weighted statistics

Research Areas:

* Zero-knowledge proofs
* Anonymous credentials
* Sybil resistance systems

Success Metric:

* Public statistics remain trustworthy without requiring real-name identification

---

## v6 — Pennsylvania Policy Database

Goal:
Create a structured policy knowledge base.

Features:

* Issue explorer
* Policy encyclopedia
* Candidate-policy links
* Supporting and opposing arguments
* Historical issue tracking

Success Metric:

* Citizens can understand issues without first choosing a candidate
